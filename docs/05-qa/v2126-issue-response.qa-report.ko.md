### SC-3 — 릴리스 스크립트 dry-run

`bash scripts/release-plugin-tag.sh --dry-run` (tail):

```
[release] canonical BKIT_VERSION=2.1.25 → tag=v2.1.25
[release] FAIL — working tree not clean. Commit or stash first.
EXIT=1
```

- 출력 어디에도 **"Path not found" 오류 없음**(`grep -niE "path not found"` → 0 매치). ✅
- non-zero 종료는 **예상된 보존** "working tree not clean" 사전 게이트(미커밋 변경 존재). ✅
- `[release][info]` 정보성 일관성 단계가 **git-tag 단계 이전에 올바르게 배치됨**(`scripts/release-plugin-tag.sh:135-142`):
  ```
  echo "[release] info: consistency check — claude plugin tag . --dry-run (informational only, never gates)"
  claude plugin tag . --dry-run 2>&1 | sed 's/^/[release][info] /' || true
  ...
  echo "[release] DRY RUN — would invoke: git tag -a ${TAG} ..."
  ```
  이번 실행에서 도달하지 못한 이유는 앞선 clean-tree 게이트가 먼저 종료되기 때문(올바른 순서).
- 정보성 명령을 직접 프로브하면 `[release][info]` 접두 출력이 나오며 **"Path not found" 없음**(미커밋 변경을 정보성으로 보고, `|| true` 로 흡수):
  ```
  [release][info] ✘ Uncommitted changes affecting this release — commit them first ...
  [release][info]   .claude-plugin/plugin.json
  [release][info]   .mcp.json     ← 추적된 삭제(relocation), 예상됨
  ...
  ```
