## Expected Output Verification

Following the expected structure and format patterns:

## HTML/CSS Prototype Quality
1. Produces a self-contained HTML file with embedded or linked CSS that renders correctly in a browser
2. Kanban board layout uses CSS grid or flexbox with four distinct columns (To Do, In Progress, In Review, Done)
3. Each column contains at least 2-3 sample task cards demonstrating the card design
4. Task cards display all required elements: title text, assignee avatar placeholder, priority badge, and due date
5. Color coding or visual indicators distinguish priority levels (high, medium, low) on each card

## Component Structure Quality
6. Sidebar component is fixed-position with project list items and filter controls (dropdowns or checkboxes)
7. Top bar spans full width with search input, notification icon with badge count, and user avatar with dropdown trigger
8. Card elements have visible grab handles, hover states, or cursor changes indicating drag-and-drop capability
9. Column headers show the column name and a task count badge
10. Visual hierarchy is clear with appropriate font sizing (headings larger than body, labels smaller than content)

## UX Pattern Quality
11. Consistent spacing and padding applied across all cards, columns, and layout sections
12. Drop zone indicators are shown (dashed borders, highlighted areas, or placeholder cards) for drag targets
13. Empty state is handled in at least one column (shows a message like "No tasks" or a dashed add button)
14. Responsive considerations exist with columns wrapping or scrolling horizontally on narrower viewports
15. Overall color palette is cohesive with no more than 5 primary colors plus neutral grays for backgrounds and borders
